# AI 工作流应用手动验收清单
## 1. 画布与编排
- 启动应用后，确认左侧“节点工坊”、中间“编排舞台”、右侧“节点观察席”都能正常显示。
- 依次点击“开始节点”“模型节点”“审核节点”“结束节点”，确认画布中会新增对应节点。
- 选中节点后修改标题、说明、模型名或审核说明，确认右侧表单可编辑，且画布中的节点内容同步变化。
- 切换浅色 / 暗色主题后，确认页面主背景、节点卡、输入区和右侧面板都会一起切换。

## 2. 工作流运行
- 使用默认流程或导入 `validation/fixtures/workflow-basic.json`，点击“运行当前工作流”，确认结果区出现“已完成”状态。
- 运行包含审核节点的流程，确认结果区变为“待审核”，右侧“待审核任务”出现新卡片。
- 在待审核任务中填写备注并点击“审核通过并继续”，确认流程继续执行，结果区最终完成。
- 在待审核任务中点击“驳回并终止”，确认流程结束且状态变为“执行失败”。

## 3. 持久化
- 修改节点位置和右侧配置后关闭应用，再重新打开，确认编排草稿会自动恢复。
- 在审核备注输入框中填写内容但不提交，关闭并重启应用，确认备注草稿仍然存在。
- 运行到审核暂停后关闭应用，再重新打开，确认“待审核任务”仍然存在。

## 4. 模型服务
- 在右侧“模型服务”面板中修改服务地址或默认模型后点击“保存模型配置”，确认日志出现保存成功提示。
- 点击“测试模型连接”，确认日志输出连接成功和响应预览。
- 故意填错服务地址后再次测试，确认日志能看到中文错误提示。

## 5. 导入导出与历史
- 导入 `validation/fixtures/workflow-document-sample.json`，确认工作流、主题和工作区都能恢复。
- 导出当前工作流 JSON，再重新导入，确认导入前后节点和配置一致。
- 运行至少一次工作流后，确认右侧“运行历史”出现记录，并能点开查看步骤详情。
- 在历史详情中导出 Markdown 和 JSON，确认文件落盘成功。

## 6. 构建与交付
- 运行 `./node_modules/.bin/tsc.cmd --noEmit`，确认类型检查通过。
- 运行 `npm run lint`，确认静态检查通过。
- 如需交付验证，运行 `npm run package`，确认 `out/flowapp-win32-x64/flowapp.exe` 存在。
